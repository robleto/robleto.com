# Runbook: Homepage Competitive Audit

## Inputs
- Target URL: `https://robleto.com`
- Competitor sample size: `N=20`

## Inventory Checklist Schema
For each homepage, capture:
- Site name + URL + category
- Headline and subhead
- Primary CTA and secondary CTA
- Top-level nav labels
- Proof markers above the fold
- Presence flags:
  - Case studies
  - Writing
  - Services positioning
  - Testimonials
  - Client logos
  - Metrics/outcomes
  - About/bio
  - Contact
  - Newsletter
  - Now/current focus
  - Process
  - Toolstack
- Scores (1-5): clarity, specificity, credibility, skimmability
- Notes

## Workflow Checklist
1. Analyze target homepage first and capture baseline row.
2. Build a diverse competitor set (`N=20`).
3. Fill inventory rows consistently.
4. Compute frequency counts by element.
5. Identify high-impact deltas (common in set, weak on target).
6. Identify differentiator patterns (rarer but high leverage).
7. Draft copy/CTA options (specific and actionable).
8. Propose experiments with impact/effort + homepage placement.

## Outputs
- Report: `research/competitive/homepage-competitive-audit.md`
- CSV: `research/competitive/homepage-inventory.csv`

## Rules
- Evidence-based claims only (quantified)
- No competitor copy-paste
- Prioritize trust, clarity, and explicit “what to do next”
- Keep recommendations split:
  - `content/structure`
  - `visual/layout`
