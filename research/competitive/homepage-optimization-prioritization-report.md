# Homepage Optimization Report: Items, Prioritization, and Layout

**Target:** [robleto.com](https://robleto.com)  
**Peer group decision:** work + writing + exploration personal sites  
**Required inspirations included:** Dan Mall, Ethan Marcotte, Brad Frost, Frank Chimero, Josh W Comeau, Maggie Appleton, Lynn Fisher  
**Evidence base:** `n=20` competitors + target baseline  
**Source files:** `research/competitive/homepage-inventory.csv`, `research/competitive/homepage-competitive-audit.md`  
**Cohort rationale appendix:** `research/competitive/homepage-peer-rationale-appendix.md`  
**Date:** 2026-03-04

## Decision Goal

Define the **optimal homepage items** to include on `robleto.com`, with clear **priority** and **layout order**, while preserving the personal-site character that this peer cohort shares.

## Cohort Signal Summary

- Writing/blog path: **20/20**
- About/bio context: **20/20**
- Case studies/work artifacts: **14/20**
- Contact path: **15/20**
- Services positioning: **7/20**
- Newsletter capture: **7/20**
- Metrics/outcomes: **3/20**
- Now/current-focus: **3/20**
- Testimonials: **0/20**
- Process module: **0/20**

## Prioritization Method

Each item is ranked by:

- **Evidence strength** (how common in this cohort).
- **Delta size** (how weak/missing on current homepage).
- **Expected impact** on confidence + action.
- **Effort**.

Priority labels:

- **P0** = implement first.
- **P1** = second wave.
- **P2** = optional.

## Recommended Homepage Item Stack

### P0 (Must include now)

1. **Hero with explicit “start here” action language**
   - `Evidence`: direct primary action appears in practice across cohort homepages.
   - `Target delta`: CTA is present but softer than peer best examples.
   - `Recommendation`: add one explicit line under subhead: “Start with selected work.”
   - `Type`: content/structure

1. **Work strip near top (3 selected artifacts)**
   - `Evidence`: work artifacts in **14/20**.
   - `Target delta`: work exists but should be surfaced sooner in scroll flow.
   - `Recommendation`: place “Selected Work” immediately after hero/supporting block.
   - `Type`: content/structure

1. **Compact trust rail (identity + recency + contact)**
   - `Evidence`: about **20/20**, contact **15/20**, writing **20/20**.
   - `Target delta`: trust signals are currently distributed.
   - `Recommendation`: cluster credibility points above fold/near fold.
   - `Type`: visual/layout

1. **How I help block (2-4 collaboration modes)**
   - `Evidence`: services explicit in **7/20** (not universal, but high leverage).
   - `Target delta`: service framing on target is implicit.
   - `Recommendation`: add concise collaboration-mode language without turning the site into an agency page.
   - `Type`: content/structure

### P1 (Second wave)

1. **Writing intro reframed as method/proof**
   - `Evidence`: writing is **20/20** baseline in this peer class.
   - `Target delta`: writing is already strong; framing can better connect it to practice.
   - `Recommendation`: rewrite section intro to position writing as how you think and ship.
   - `Type`: content/structure

1. **Optional recurrence module (Now or newsletter)**
   - `Evidence`: newsletter **7/20**, now **3/20**.
   - `Target delta`: no dedicated recurrence module on homepage.
   - `Recommendation`: choose one based on current goal (audience growth vs recency signal).
   - `Type`: content/structure

### P2 (Optional)

1. **Metrics chips / logo snippets**
   - `Evidence`: metrics **3/20**, logos **2/20**.
   - `Target delta`: currently absent.
   - `Recommendation`: add only if data is specific and links to concrete work proof.
   - `Type`: visual/layout

1. **Process strip**
   - `Evidence`: explicit process **0/20** in this cohort.
   - `Target delta`: absent, but not expected baseline.
   - `Recommendation`: include only if it improves clarity for your intended clients.
   - `Type`: content/structure

## Recommended Layout Order (Top → Bottom)

1. Hero (identity + value + primary/secondary CTA)
1. Trust rail (role context + recency + contact)
1. Selected Work (3 highlighted artifacts)
1. How I help (2-4 collaboration modes)
1. Signals (keep)
1. Writing (keep, reframed intro)
1. Influences (keep)
1. Optional footer module (Now or newsletter)

## Why this is optimal for your peer class

- It preserves your strongest differentiator: exploratory editorial depth.
- It aligns with how peers communicate credibility (writing + artifacts + clear identity).
- It avoids over-indexing on agency conventions that this cohort largely does not use.

## Final Recommendation

Implement these first: **(1) explicit “Start here” CTA language, (2) top-of-page Selected Work strip, (3) compact trust rail.** Then add **How I help** as the conversion bridge into your deeper exploratory sections.
